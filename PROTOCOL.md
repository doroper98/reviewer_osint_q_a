# Reviewer ↔ OSINT 교신 프로토콜 (v1)

`agents_reviewer`(보고서·`report_bundle` **producer**)와 `osint_generator`(영상
**consumer**) 사이의 계약·질문·통지를 **비동기 파일 기반**으로 주고받는 규칙.
목적: 사람이 두 저장소 사이에서 메시지를 복붙 중계하지 않게 한다. 이 저장소가
교신의 SSOT(단일 출처)다.

## 참여자 (sender 코드)

| 코드 | 주체 |
|------|------|
| `agent_reviewer_bot` | agents_reviewer 쪽 에이전트 (질문·통지 발신) |
| `osint_generator`    | osint_generator 쪽 에이전트/담당자 (질문·통지 발신) |

양쪽 다 발신자가 될 수 있다. 규칙은 대칭이다.

## 핵심 규칙

1. **한 사안 = 한 스레드 파일.** osint(또는 reviewer)에게 전달·질문·통지할 게
   생기면 **항상 새 파일**을 만든다. 기존 파일에 다른 사안을 얹지 않는다.
2. **파일명 코드 체계** — `yyyy_mm_dd_hhmmss_<sender>_q_nn.md`
   - `<sender>` = `agent_reviewer_bot` 또는 `osint_generator` (발신자).
   - `nn` = 발신자별 2자리 일련번호(`01`, `02`, …). 발신자마다 독립 카운터.
   - `yyyy_mm_dd_hhmmss` = 생성 시각(KST).
   - 예: `2026_07_09_110842_agent_reviewer_bot_q_01.md`.
3. **파일 위치** — 모든 스레드는 `threads/` 디렉토리에 둔다.
4. **한 파일 = 한 스레드.** 같은 파일 안에서 질문 → 답변 → 조치·종결을 이어 쓴다.
   - 발신자가 `## 질문 / 전달` 을 채운다.
   - 수신자가 `## 답변` 을 채운다.
   - 사안이 끝나면 **발신자가** `## 조치 · 종결` 에 *무엇을 어떻게 조치했고*
     *언제 종결했는지* 를 기록하고 `status: closed` 로 바꾼다.
5. **답변에 새 요청을 끼워넣지 않는다.** 답변·종결을 쓰다가 새로 부탁·질문할 게
   생기면, 그 답변에 넣지 말고 **새 스레드 파일**(규칙 1·2)을 만든다. 한 스레드는
   원래의 한 사안만 다룬다. (교차 참조는 `refs` 로 건다.)
6. **상태(status) 라이프사이클** — 헤더 메타의 `status` 를 갱신한다:
   `open`(답변 대기) → `answered`(답변 옴, 조치 진행 중) → `closed`(조치 완료·종결).
   필요 시 `reopened` → `answered` → `closed` 반복.
7. **교차 참조(`refs`)** — 다른 스레드/계약 조항/버전과 얽히면 파일명·`§조항`·
   버전(예: `IMAGE_BUNDLE_CONTRACT §3.1-a`, `producer v8.3.6`)을 `refs` 에 적는다.
8. **커밋** — 각자 자기 쪽 변경만 커밋·푸시한다. 커밋 메시지에 스레드 파일명(또는
   `q_nn` 코드)을 포함한다. 상대 스레드 파일을 임의로 수정하지 않는다(자기 섹션만).
9. **한 사안 종결의 정의** — 발신자가 답변을 받아 실제 조치(코드/문서/버전 반영
   또는 "조치 불요" 판단)를 완료하고 `## 조치 · 종결` + `status: closed` 를 기록한
   시점. 답변만 오고 종결이 없으면 스레드는 계속 열려 있다.

## 파일 템플릿

`templates/question_template.md` 를 복사해 파일명 규칙(규칙 2)대로 저장한 뒤 채운다.

## 헤더 메타 필드

| 필드 | 의미 |
|------|------|
| `id` | 파일명(확장자 제외)과 동일한 스레드 코드 |
| `from` / `to` | 발신자 / 수신자 코드 |
| `created` | 생성 시각 (KST) |
| `status` | `open` / `answered` / `closed` / `reopened` |
| `refs` | 관련 계약 조항·버전·다른 스레드 파일명 |

## 이 프로토콜 자체의 변경

이 문서(`PROTOCOL.md`)가 규칙의 SSOT다. 변경 시 `agents_reviewer` 의
`CLAUDE.md` "OSINT 교신 규칙" 섹션과 **정합**을 맞춘다(양쪽에 요약이 있으나 상세
규칙의 정본은 이 파일). 규칙 변경도 하나의 스레드로 제안·합의 후 반영한다.
